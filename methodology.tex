\section{Executive Summary}
% ============================================================

Zoiro Broast Hub is a comprehensive restaurant management system that enables online food ordering, kitchen operations, delivery management, and complete business administration. The system supports multiple user roles with role-based access control (RBAC) to ensure secure and efficient operations.

\subsection{Key Features}
\begin{itemize}
    \item Online food ordering with real-time tracking
    \item Multi-role employee portal
    \item Kitchen display system
    \item Delivery rider management
    \item Table reservation and dine-in service
    \item Loyalty program with tiered rewards
    \item Inventory and payroll management
    \item Comprehensive reporting and analytics
\end{itemize}

The system follows an \textbf{Agile Development Methodology} with iterative sprints focusing on continuous delivery and user feedback integration.

\subsubsection{Architecture Pattern}

The system implements a \textbf{Multi-Layer Architecture}:

\begin{enumerate}
    \item \textbf{Presentation Layer:} React components with server-side rendering
    \item \textbf{Business Logic Layer:} Server actions and API routes
    \item \textbf{Data Access Layer:} Supabase client with RPC functions
    \item \textbf{Database Layer:} PostgreSQL with Row Level Security (RLS)
\end{enumerate}

\section{Activity Diagram}
% ============================================================

The Activity Diagram shows the complete order flow process involving all 7 roles.

\subsection{Diagram Overview}

\begin{center}
\textit{[Insert activity-diagram.puml rendered image here]}
\end{center}

\subsection{Process Phases}

\subsubsection{Phase 1: Customer Ordering}
\begin{enumerate}
    \item Customer visits website
    \item Browses menu categories
    \item Views item details
    \item Adds items to cart
    \item Applies promo code (optional)
    \item Selects order type (Delivery/Walk-in/Dine-in)
    \item Chooses payment method
    \item Completes checkout
\end{enumerate}

\subsubsection{Phase 2: Order Processing}
\begin{enumerate}
    \item System generates order number
    \item Email confirmation sent
    \item Push notification sent
    \item Kitchen staff alerted
    \item Manager monitors dashboard
\end{enumerate}

\subsubsection{Phase 3: Kitchen Preparation}
\begin{enumerate}
    \item Kitchen receives order alert
    \item Checks ingredient availability
    \item Starts cooking (status: PREPARING)
    \item Customer receives notification
    \item Completes cooking
    \item Quality check performed
    \item Marks order as READY
\end{enumerate}

\subsubsection{Phase 4: Order Fulfillment}

\textbf{A) Home Delivery:}
\begin{enumerate}
    \item System assigns nearest rider
    \item Rider accepts delivery
    \item Picks up order from restaurant
    \item Status updated: ON THE WAY
    \item Customer tracks rider on map
    \item Rider arrives and delivers
    \item Collects cash if applicable
    \item Marks as DELIVERED
\end{enumerate}

\textbf{B) Walk-in Pickup:}
\begin{enumerate}
    \item Billing staff announces order
    \item Customer presents confirmation
    \item Payment processed (if cash)
    \item Receipt printed
    \item Order handed over
\end{enumerate}

\textbf{C) Dine-in Service:}
\begin{enumerate}
    \item Waiter retrieves order from kitchen
    \item Serves at customer's table
    \item Customer enjoys meal
    \item Bill requested
    \item Billing staff calculates total
    \item Payment processed
    \item Receipt provided
    \item Waiter clears table, records tips
\end{enumerate}

\subsubsection{Phase 5: Post-Order Activities}
\begin{enumerate}
    \item System calculates loyalty points
    \item Points awarded to customer
    \item Customer may write review
    \item System updates analytics
    \item Manager reviews reports
    \item Inventory levels checked
    \item Low stock alerts generated
\end{enumerate}

\subsubsection{Phase 6: Admin Activities}
\begin{enumerate}
    \item Admin views system activity
    \item Checks audit logs
    \item Manages employee accounts
    \item Processes monthly payroll
    \item Approves payments
    \item Performs database backup
\end{enumerate}

% ============================================================

The Class Diagram represents all business entities and their relationships.

\subsection{Diagram Overview}

\end{center}

\subsection{Entity Groups}

\subsubsection{People Entities}

\textbf{Customer}
\begin{itemize}
    \item Attributes: Name, Email, Phone, Address
    \item Loyalty: Points, Tier (Bronze/Silver/Gold/Platinum)
    \item History: Orders, Favorites, Reviews
\end{itemize}

\textbf{Employee}
\begin{itemize}
    \item Identification: Employee ID, Name, Email, Phone
    \item Work Info: Role, Status, Salary, Hire Date
    \item Security: 2FA, Permissions, Last Login
    \item Performance: Tips, Orders Taken
\end{itemize}

\textbf{Employee Roles}
\begin{itemize}
    \item Admin - Full system access
    \item Manager - Operations management
    \item Waiter - Table service
    \item Kitchen Staff - Food preparation
    \item Billing Staff - Payment processing
    \item Delivery Rider - Order delivery
    \item Other - Custom permissions
\end{itemize}

\subsubsection{Menu Entities}

\textbf{Category}
\begin{itemize}
    \item Attributes: Name, Description, Display Order, Image
    \item Examples: Broast Chicken, Burgers, Drinks, Desserts
\end{itemize}

\textbf{Menu Item}
\begin{itemize}
    \item Basic: Name, Description, Price
    \item Details: Prep Time, Availability, Featured
    \item Ratings: Average Rating, Review Count
    \item Variants: Size Options, Size Prices
\end{itemize}

\textbf{Deal / Combo}
\begin{itemize}
    \item Pricing: Original Price, Discounted Price, Savings
    \item Contents: Included Items List
    \item Validity: Available Until Date
\end{itemize}

\subsubsection{Order Entities}

\textbf{Order}
\begin{itemize}
    \item Identification: Order Number, Date/Time
    \item Type: Delivery, Walk-in, Dine-in
    \item Pricing: Subtotal, Discount, Delivery Fee, Tax, Total
    \item Status: Pending → Confirmed → Preparing → Ready → Delivering → Delivered
\end{itemize}

\textbf{Order Item}
\begin{itemize}
    \item Reference: Menu Item
    \item Details: Quantity, Size, Price, Instructions
\end{itemize}

\textbf{Invoice}
\begin{itemize}
    \item Identification: Invoice Number
    \item Payment: Total, Method, Status, Date
\end{itemize}

\textbf{Payment Types}
\begin{itemize}
    \item Cash, Card, Online, Wallet
\end{itemize}

\subsubsection{Operations Entities}

\textbf{Restaurant Table}
\begin{itemize}
    \item Physical: Table Number, Capacity, Floor/Section
    \item Status: Available, Occupied, Reserved, Cleaning
    \item Assignment: Current Order, Assigned Waiter
\end{itemize}

\textbf{Inventory Item}
\begin{itemize}
    \item Stock: Name, Quantity, Min Level, Unit
    \item Supplier: Name, Cost per Unit
    \item Alerts: Last Restocked, Low Stock Flag
\end{itemize}

\textbf{Attendance}
\begin{itemize}
    \item Record: Employee, Date
    \item Time: Check-in, Check-out
    \item Calculation: Hours Worked, Overtime
    \item Status: Present, Absent, Late, On Leave
\end{itemize}

\textbf{Payroll}
\begin{itemize}
    \item Period: Employee, Month/Year
    \item Earnings: Base Salary, Tips, Bonus
    \item Deductions: Various deductions
    \item Result: Net Amount, Paid Status
\end{itemize}

\subsubsection{Marketing Entities}

\textbf{Special Offer}
\begin{itemize}
    \item Discount: Type, Value, Min Order, Max Discount
    \item Period: Valid From/Until
    \item Display: Show Banner, Send Push Notification
\end{itemize}

\textbf{Loyalty Tiers}
\begin{itemize}
    \item Bronze: 0-500 points
    \item Silver: 501-1,500 points
    \item Gold: 1,501-3,000 points
    \item Platinum: 3,001+ points
    \item Benefits: Points per Rs.100, Birthday Bonus, Free Delivery
\end{itemize}

\textbf{Review}
\begin{itemize}
    \item Content: Customer, Order, Rating (1-5), Comment
    \item Meta: Date, Visibility
\end{itemize}

\textbf{Notification}
\begin{itemize}
    \item Target: Recipient
    \item Type: Order Update, Promo, Alert
    \item Content: Title, Message, Read Status
\end{itemize}

\subsubsection{Security Entities}

\textbf{Audit Log}
\begin{itemize}
    \item Tracking: Timestamp, User, IP Address
    \item Action: Type (Create/Update/Delete/View)
    \item Data: Entity, Before/After Values
\end{itemize}

\textbf{Session}
\begin{itemize}
    \item Authentication: User, Token
    \item Details: Device, Login Time, 2FA Verified
\end{itemize}
